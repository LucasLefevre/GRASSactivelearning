NAME

activelearning.py - Classification of raster areas with active learning.

KEYWORDS

classification machine learning scikit-learn

SYNOPSIS

activelearning.py
activelearning.py --help
activelearning.py training_set=string test_set=string unlabeled_set=string [learning_steps=integer] [diversity_select_from=integer] [diversity_lambda=float] [c_parameter=float] [gamma_parameter=float] [--help] [--verbose] [--quiet] [--ui]

Flags:

--help
Print usage summary
--verbose
Verbose module output
--quiet
Quiet module output
--ui
Force launching GUI dialog

Parameters:

training_set=string [required]
Training set (csv file)
test_set=string [required]
Test set (csv file)
unlabeled_set=string [required]
Unlabeled samples (csv file)
Default:
learning_steps=integer
Number of samples to label at each iteration
diversity_select_from=integer
Number of samples to select (based on uncertainty criterion) before applying the diversity criterion.
diversity_lambda=float
Lambda parameter used in the diversity heuristic
c_parameter=float
Penalty parameter C of the error term
gamma_parameter=float
Kernel coefficient

DESCRIPTION

This module uses SVM from the scikit learn python package to perform classification on regions of raster maps. These regions can be the output of i.segment or r.clump.

The module enables learning with only a small initial labeled data set via active learning. This semi-supervised learning algorithm interactively query the user to label the regions that are most useful to improve the overall classification score. With this technique, the number of examples to learn the classification is often much lower than the number of examples needed in normal supervised algorithm.

Data for the training set, test set and the unlabeled set should be in a file in csv format (for example coming from the i.segment.stats module) with features for every region. A header containing the feature's name should be the first line of the file and all examples should be uniquely identified by the first column. The classes for the training and test examples should be the second column.

NOTES

This module requires the scikit-learn python package. This module needs to be installed in your GRASS GIS Python environment. Please refer to r.learn.ml's Notes to install this package.
